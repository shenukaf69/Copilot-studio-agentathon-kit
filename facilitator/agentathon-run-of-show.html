Before the room opens

  • Every attendee has a Copilot Studio licence or trial, and maker permission on the target environment
  • Each team has downloaded the .xlsx for their chosen scenario (see table below) before step 3 — don't burn live-build minutes on file hunting
  • Dataverse / Dynamics 365 / Power Automate connectors pre-authorized in the environment — first-time connector consent mid-build eats 5+ minutes per team
  • Teams admin is aware a Teams-channel publish request may land during or after the session — approval will not complete inside the 90 minutes
  • Decide up front: are teams publishing for real, or is "green in the Test pane" the finish line? Say this in the kickoff so nobody stalls on step 10

The 10-step build (identical for every scenario)

This is the exact sequence used across all 12 scenario PDFs — only the agent name, data file, topic, and tool change. Demo each step live on your own scenario; teams mirror it on theirs in the same window.

Part 1 — Set up
01

Create the agent

copilotstudio.microsoft.com → Create → New agent. Set name, description, solution/environment, icon.

02

Choose orchestration mode

Generative (AI-led, recommended for this session) vs Classic (trigger-led, for regulated flows). Settings → Generative AI.

03

Add knowledge

Knowledge tab → Add knowledge → File → upload the scenario's sample data workbook.

Part 2 — Behavior
04

Add agent instructions

Overview → Instructions. Name the audiences, set the source-of-truth rule, the tool-triggering rule, the data-boundary rule.

05

Add a custom topic

Topics → Add a topic → Create from description. Name, trigger phrases, and a numbered conversation flow: identify → collect → look up → call tool → confirm.

Part 3 — Action & governance
06

Add a tool

Prebuilt connector, Power Automate flow, custom HTTP, or MCP server. Name it, describe it, list its inputs and its outcome.

07

Configure settings

General, Generative AI (moderation: start Medium), Authentication ("Authenticate with Microsoft"), Security (DLP, channels).

Part 4 — Quality & ship
08

Evaluate

Analytics → Evaluations → New evaluation. Pair prompts with expected outcomes; score Groundedness, Relevance, Accuracy.

09

Test in the Test pane

Activity map shows which topic/tool/knowledge fired, what it collected, and citations back to the source rows.

10

Publish

Teams, Microsoft 365 Copilot, custom web embed, or Slack / Facebook / Telephony. Teams & M365 Copilot need tenant admin approval first.

Try these live (say this at minute 75)

Analytical prompt

"Show me [the scenario's key metric] by [its main grouping], highest first." — proves knowledge grounding and math-over-rows.

Risk / filter prompt

"Which [records] are [overdue / escalated / breaching] right now?" — proves the agent reasons over dates and status fields.

Governed action

The scenario's own topic trigger, e.g. "Escalate case CS4007" or "Move Cobalt Insurance to Negotiate" — proves the tool call fires end-to-end.

Audit check

"List the [actions] I triggered this session" — proves the action left a real trail, not just a chat reply.

Facilitator pitfalls

Orchestration mode debate eats step 2

Don't let teams deliberate. State up front: "everyone uses Generative Orchestration today" — Classic is worth a one-line mention, not a detour.

First-time connector consent

Dataverse / D365 / Planner / Azure DevOps connectors ask for account consent the first time a maker touches them. Pre-authorize in the environment before the session, or budget 5 minutes at step 6.

Publish ≠ finish line

Teams and M365 Copilot channels need tenant admin approval that will not land in 90 minutes. Set the expectation at kickoff: the Test pane, not a live Teams channel, is what gets demoed at minute 90.
